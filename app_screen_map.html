Center content — changes based on sidebar selection

Screen 3 — Lab order dispatch
Lab order dispatch

Screen 4 — Referral + shift handover
Referral + shift handover

Private Doctor — الطبيب الخاص
Same Z-layout shell. Two permission modes — patient-present (biometric gated) and patient-absent. Doctor identities masked from each other. 3 main screens.
The two modes share the same layout shell but render different data. The biometric gate screen appears before patient-present mode activates.
Screen 1 — Biometric gate (before patient-present activates)
Identity gate Security
  • Patient name + ID displayed (doctor selected patient from their list)
  • Instruction: "Ask patient to confirm fingerprint"
  • Biometric confirmation widget (fingerprint reader integration point)
  • Doctor professional ID re-confirmation field
  • Both confirmed → patient-present mode unlocks
  • Either fails → falls back to patient-absent mode, gate event logged
  • Gate event written to case_entries (immutable) with timestamp

Patient-present clinical view Biometric active

Medical Staff — الفريق الطبي / التمريض
Ward operations only. No clinical write access. Per-bed view with live vitals, scheduled meds, and admission data. 3 main screens.
Screen 1 — Ward bed grid (default view)
Ward bed grid Live
  • Ward name + current occupancy count
  • On-duty doctors list (who is signed in, which ward)
  • Shift handover summary card (from outgoing shift)
Bed card grid — each bed card contains
  • Patient name + national ID
  • Admission date + elapsed stay duration
  • Scheduled medications for current shift (derived from active prescriptions — name, dose, time)
  • Live vitals: BP, heart rate, temperature, SpO2 (most recent reading) Live
  • Bed status: occupied / vacant / pending discharge
  • Escalation flag button (one-tap → fires undismissable banner to assigned doctor)
  • ─── If vacant: quick-admit button
  • ─── If pending discharge: discharge checklist progress indicator
Screen 2 — Case entry log + vitals entry
Case log + vitals entry
Left — log feed
  • Append-only event feed
  • Server timestamps
  • Filter by type / date
  • Types: observation, note, escalation, procedure, event
Right — entry forms
  • New case entry form (select type, free text, auto-timestamps)
  • ─────
  • Vitals entry per patient: BP, HR, temp, SpO2, RR, weight
  • Select patient from ward list
  • Submit → writes to patient vitals + updates bed card live
  • ─────
  • Supply request form (item, quantity, urgency, ward)
Screen 3 — Admission + discharge flow
Admission + discharge
  • Admission form: search patient by national ID, select bed, set type (emergency / scheduled / transfer), assign primary doctor from on-duty list → submit writes case_entry + updates bed grid
  • Discharge checklist: medication given, patient briefed, discharge summary printed, bed cleared — all items must be ticked before bed resets to vacant
  • Read-only access: active prescriptions (to administer meds), CRs (for clinical context — cannot write)
Administration — الإدارة
Civil and billing data only. Zero clinical access. No path from this interface to any medical record. 3 main screens.
Screen 1 — Patient civil registry
Civil patient registry
  • Search by name / national ID / admission date range
  • Total admissions counter (today / this month)
  • Current ward occupancy rates
Patient civil record (clicking a search result opens this — no clinical fields present)
  • First name, last name, national ID, date of birth, gender
  • Address: wilaya + commune + phone + emergency contact
  • Admission history: date in, date out, ward, admission type — NO diagnosis, NO reason
  • Billing records linked to each admission
  • Document generation: admission certificate, discharge cover sheet (civil header only), billing statement
Screen 2 — Billing management
Billing management
  • Billing entry list: patient name, admission ref, date, service category (consultation / lab / imaging / bed day / pharmacy — category only, no drug names), amount, status
  • Create new billing entry (linked to admission)
  • Mark as paid / pending / waived
  • Export billing statement PDF per patient
  • Monthly revenue summary (totals by category)
Screen 3 — Operational statistics
Operational stats
  • Total admissions this month per ward
  • Average length of stay per ward
  • Current occupancy rate per ward (live) Live
  • Discharge rate this month
  • Supply requests pending vs fulfilled
  • No clinical metrics — no diagnoses, outcomes, or mortality data shown

Orders queue + result entry Live

Order queue

Result entry panel (selected order)
